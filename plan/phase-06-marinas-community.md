# Фаза 6 — Порты, марины, якорные стоянки, сообщество

**Цель:** справочник POI (причалы, услуги, контакты); карта якорных стоянок с атрибутами (защита от ветра, тип дна — по данным); отзывы и опционально бронирование; после стабилизации ядра — Crew Match и лента путешествий.

**Зависимости:** [Фаза 1](phase-01-navigation-core.md); для синхронизации отзывов — бэкенд или BaaS (решение в этой фазе).

**Ссылки:** **[`../Подробный план реализации.md`](../Подробный%20план%20реализации.md)** (модуль M6, Приложение A).

---

## Связь с финальным планом

- **TDD:** тесты синхронизации каталога (разрешение конфликтов), очереди отзывов офлайн, Crew Match без сети.
- **Аудит:** CRUD маркеров якорной стоянки (`mooring_marker_*`), публикация отзыва, отправка Crew Match — только идентификаторы в `context_json`, без свободного текста сообщений (минимизация PII).
- **Логирование:** модерация/репорты — структурированные события без копирования пользовательского контента в логи.

## Шаги и артефакты

| Шаг | Действие | Выход |
|-----|----------|--------|
| 6.1 | Модель данных марина/порт/якорная; импорт из открытых или лицензированных каталогов | SQLite + синхронизация пакетов |
| 6.2 | Карточка объекта: контакты, VHF, услуги (электричество, вода), ссылки | UI |
| 6.3 | Отзывы: локальный черновик офлайн, пост-модерация при наличии сервера | API + conflict policy |
| 6.4 | Бронирование: интеграция партнёра или deeplink — по наличию договорённостей | Опционально |
| 6.5 | Crew Match / лента: минимальный MVP (объявления рейсов, профили) — приоритет продукта | Отдельный спека |

## Детализация

### Offline-first

- Отзывы: очередь исходящих при восстановлении связи.

## Критерии приёмки

- Пользователь находит марину на карте и открывает карточку.
- Отзыв можно оставить офлайн и дослать позже (если включена синхронизация).
