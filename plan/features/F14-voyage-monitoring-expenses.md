# F14 — Мониторинг рейса, расходы и «берег знает» (класс SafeTrx / SailGrib WR)

**Источник идей:** SafeTrx (отслеживание позиции, оповещение береговой охраны при пропаже); SailGrib WR (погода + маршруты + **бортовой журнал расходов**); упоминания расходов в портах через Navily в [`../../IDEAS.md`](../../IDEAS.md).

## Связь с фазами

- [phase-07-safety-logbook.md](../phase-07-safety-logbook.md) — журнал, SOS, треки.
- Погода/маршруты — [F02](F02-weather-forecast-animation.md), [F10-offline-grib.md](F10-offline-grib.md).

## План реализации фичи

### Мониторинг рейса (SafeTrx-подобный сценарий)

1. Регистрация плана рейса: точки отправления/прибытия, ETA окно, количество людей на борту.
2. Периодическая отправка позиции на выбранный контакт или партнёрский SAR API — **только с явным согласием и региональной проверкой юридики**.
3. Неудачные проверки-in («пропали с радаров») → эскалация по настройке пользователя (SMS/email/push контактам).
4. Отдельный режим от SOS: здесь «я в порядке, просто отчитываюсь».

### Учёт расходов

1. Статьи: топливо, марины, провизия, ремонт; валюта поездки.
2. Привязка расхода к марине/порту из [F05](F05-marinas-anchorages-booking.md).
3. Экспорт CSV для совместимости с домашней бухгалтерией.

## Критерии приёмки

- Пользователь может вести журнал расходов офлайн и выгрузить CSV.
- Любая передача позиции третьим лицам требует включённой настройки и понятного описания политики приватности.

## Риски

| Риск | Митигация |
|------|-----------|
| Юридические обязательства SAR-партнёров | Интеграции только после юр. ревью региона |
| Ложные тревоги | Таймеры подтверждения «всё ок» |
