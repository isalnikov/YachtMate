# F03 — Приливы, графики, астрономия (класс Tides Planner + общий блок приливов)

**Источник идей:** Tides Planner; Tide Guide; Tide Times; также приливы как часть Navionics в [`../../IDEAS.md`](../../IDEAS.md).

## Возможности референса

| Источник в IDEAS | Фишки |
|------------------|--------|
| Tides Planner | Графики HW/LW, минимальная осадка для расчётов, фазы луны, восходы/закаты; атлас течений для части регионов |
| Tide Guide | Таблицы и кривые уровня; синхронизация с **умными часами**: время приливов, ветер на запястье |
| Tide Times | Выбор станции на карте; краткий **цифровой** вывод без тяжёлой инфографики — режим «просто цифры» |
| Общее | Единое место для планирования выхода по воде и по свету |

## Связь с фазами

## План реализации фичи

1. **Источник приливов:** NOAA CO-OPS (где применимо), WorldTides или аналог — выбор по региону с ADR.
2. **Модель:** станция прилива как сущность с координатами; кэш таблицы HW/LW на сутки вперёд.
3. **UI:** график высоты от времени; маркеры High/Low; переключение даты.
4. **Астрономия:** восход/заход солнца/луны через астрономическую библиотеку или API; фазы луны — расчёт или данные источника.
5. **Связка с навигацией:** быстрый переход «текущая позиция → ближайшие станции» (после геолокации).
6. **Режим Tide Times:** упрощённая карточка «разница уровня между экстремумами» для слабых приливов.
7. **Носимые устройства:** экспорт ближайших HW/LW в виджет/OS companion — после MVP (Apple Watch / Wear OS с ограничениями платформы).

## Критерии приёмки

- Для тестовой станции значения HW/LW совпадают с эталоном источника на выборке дат.
- Офлайн: последняя загруженная часть прогноза отображается с предупреждением о устаревании.
