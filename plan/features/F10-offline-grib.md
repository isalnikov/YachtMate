# F10 — Офлайн GRIB: импорт и просмотр (класс PocketGrib / ZyGrib / SailGrib)

**Источник идей:** рекомендации PocketGrib (телефон) и ZyGrib (ПК) для загрузки и изучения прогноза **без постоянного интернета**; SailGrib WR как комбинированный инструмент в [`../../IDEAS.md`](../../IDEAS.md).

## Зачем отдельная фича

Погодные API ([F02](F02-weather-forecast-animation.md)) удобны у берега; в море критична возможность заранее скачать **файл GRIB** и анализировать слои в офлайне.

## Связь с фазами

- [phase-04-weather-tides.md](../phase-04-weather-tides.md).
- Пересечение с маршрутизацией SailGrib — см. [F14-voyage-monitoring-expenses.md](F14-voyage-monitoring-expenses.md).

## План реализации фичи

1. **Импорт файла:** загрузка `.grib` / `.grb2` из файловой системы, share intent, облако (опционально).
2. **Парсинг:** библиотека чтения GRIB (нативная или Dart через FFI); отображение сетки на карте или отдельном экране с выбором переменной (ветер, давление, волны при наличии в файле).
3. **Время:** слайдер по forecast hour; синхронизация с маршрутом рейса (ссылка на точки из F01).
4. **Объём и производительность:** предупреждение о размере файла; упрощённый рендер на больших областях.
5. **Источники файлов:** не генерировать GRIB самим на MVP — пользователь приносит файл с известных сервисов; документация «как получить GRIB».

## Критерии приёмки

- Заранее загруженный файл открывается в авиарежиме и показывает хотя бы поле ветра.
- Ошибочный файл показывает понятное сообщение, без краша.

## Риски

| Риск | Митигация |
|------|-----------|
| Лицензирование данных внутри GRIB | Пользователь отвечает за источник; дисклеймер в приложении |
| Размер библиотеки | Опциональный модуль или dynamic delivery |
