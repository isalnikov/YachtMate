# F07 — Суда в реальном времени и справка по целям (класс Marine Traffic)

**Источник идей:** Marine Traffic; альтернативы Ship Finder Lite, Vessel Finder, Ship Tracker в [`../../IDEAS.md`](../../IDEAS.md).

## Возможности референса

- Карта судов с AIS по всему миру в реальном времени (береговые и спутниковые агрегаторы).
- Поиск судна по имени; карточка с характеристиками, порт приписки, флаг и т.д.
- **Marine Traffic (IDEAS):** AR-режим опознания судна камерой; **метеокарта** с обновлением ~48 ч; мониторинг портов/история.
- **Find Ship / VesselFinder и аналоги:** поиск по MMSI, IMO, позывным; измерение **расстояния между целями** жестами; фильтры по типам судов.
- **Cruise Mapper / MyShipTracking:** расширенная справка, история судна; **расписание судов в порту** на ближайшее время.
- Платные подписки у части сервисов; большой функционал.

## Связь с фазами

- [phase-03-ais-nmea.md](../phase-03-ais-nmea.md) — приём AIS через собственный NMEA/AIS.

## Различие архитектур

| Режим | Описание |
|-------|----------|
| **Локальный AIS** | Антенна/шлюз на борту — как в Фазе 3, CPA/TCPA. |
| **Глобальный трек** | Как Marine Traffic — требуется поток данных с береговых/AIS-сателлит провайдеров (коммерческий API, лицензия). |

## План реализации фичи

1. **MVP:** полноценная реализация локального AIS (Фаза 3) + отображение и поиск в пределах получаемых целей.
2. **Расширение «глобальный слой»:** опциональный облачный провайдер (контракт, ключ API, тарифы).
3. **Поиск:** локальный индекс по MMSI/name из текущих целей; глобальный поиск по MMSI/IMO/name — через облако при наличии.
4. **Инструменты карты:** линейка расстояния между двумя целями / целью и своей позицией.
5. **AR-идентификация:** пост-MVP; требует калибровки камеры и этики приватности — только как дополнение к облачному тарифу.
6. **Метеослой поверх судоходства:** переиспользовать погодный модуль ([F02](F02-weather-forecast-animation.md)), не смешивать источники без подписи.
7. **Юридика:** условия поставщика AIS и ограничения по распространению данных.

## Критерии приёмки

- Локальный поток NMEA воспроизводит те же цели, что и эталонный декодер на тест-логах.
- Облачный режим не включён без явной настройки пользователя и принятия условий провайдера.
