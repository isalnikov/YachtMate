# F01 — Морские карты, маршруты, синхронизация (класс Navionics)

**Источник идей:** Navionics Boating; также iNavX, Plan2Nav, Transas iSailor (кросс-проверка маршрутов), блоки про «яхта-навигатор», интеграцию с оборудованием в [`../../IDEAS.md`](../../IDEAS.md).

## Возможности референса

| Возможность | Описание |
|-------------|----------|
| Детальные карты по акваториям | Региональные пакеты, обновления |
| Маршруты | Прокладка, сохранение, расстояние до точек; авто-/предложение маршрута (Navionics marketing — advisory, не автопилот) |
| Погода / приливы | Встроенные или связанные модули («всё в одном экране») |
| Комментарии, метки, фото на карте | Crowdsourcing; персональные заметки |
| Кросс-платформа | Смартфон, планшет, один аккаунт; опционально веб |
| Интеграция с приборами | Вывод данных глубины/ветра/SOG на экран при наличии протоколов (абстракция Hardware) |
| Монетизация | Подписка по регионам (разные ценовые уровни) |
| Офлайн навигация | Интернет для загрузки; ход по GPS |
| Риски данных | Неточные глубины; ошибки береговой линии / «трек по суше»; региональные ограничения обновлений у части пользователей |
| Альтернативные приложения | iNavX — несколько источников карт (концепция модулей карт); iSailor — «второе мнение» для прокладки у специалистов |

## План реализации фичи

1. **Маршруты:** модель waypoints + polyline; список сохранённых маршрутов; расчёт дистанции по сегментам и до курсора (ETA опционально позже).
2. **Регионы и подписка:** абстракция «лицензия региона» в домене; без привязки к конкретному платёжному провайдера на MVP — флаги «регион разблокирован» из backend или локально для разработки.
3. **Провайдеры карт (расширение):** интерфейс «ChartProvider» для нескольких лицензируемых источников (аналог идеи iNavX); переключение слоя без потери собственных маршрутов.
4. **Интеграция приборов:** опциональный поток NMEA для отображения SOG/STW/depth/wind рядом с картой (пересечение с Фазой 3 — единый парсер).
5. **Синхронизация:** учётная запись и синк маршрутов/меток между устройствами — после офлайн-first локального хранения (конфликты по версии сущности).
6. **Комментарии и медиа на карте:** точка + текст + timestamp + автор; фото как вложение при политике хранения; модерация и офлайн-очередь — см. Фазу 6.
7. **Прозрачность рисков:** экран «ограничения данных» и напоминание сверять с официальными источниками.
8. **Автоматическая прокладка:** только после [phase-05-auto-guidance.md](../phase-05-auto-guidance.md); здесь — UI-крючки «предложить маршрут» с дисклеймером advisory.

## Критерии приёмки

- Сохранённый маршрут воспроизводится офлайн.
- Пользователь видит дисклеймер о неофициальных глубинах и некорректных треках в отдельных районах.
- Без активной подписки регион недоступен для загрузки (или режим demo-области).
