# Фаза 4 — Погода, приливы, оверлей на карте

**Цель:** метеопрогнозы с офлайн-кэшем по региону и времени; приливы и отливы; визуализация ветра/давления на карте (упрощённый Windy-like); опционально push при ухудшении условий.

**Зависимости:** [Фаза 1](phase-01-navigation-core.md); наличие bbox/региона для запросов.

**Ссылки:** [`../TODO.md`](../TODO.md), **[`../Подробный план реализации.md`](../Подробный%20план%20реализации.md)** (модули M4–M5, TTL кэшей).

---

## Связь с финальным планом

- **TDD:** тесты политики TTL кэша погоды и приливов; корректное поведение офлайна (данные из кэша с меткой времени).
- **Аудит:** явное действие пользователя «обновить прогноз» — событие с типом вроде `weather_refresh`; фоновый poll без избыточных записей в `user_action_audit`.
- **Конфиденциальность:** ключи внешних API — только через `--dart-define` / secure storage (финальный план §10).

## Шаги и артефакты

| Шаг | Действие | Выход |
|-----|----------|--------|
| 4.1 | Клиент Open-Meteo или иного API; модели данных; кэш в SQLite с TTL и ключом регион+время | Пакет `weather` |
| 4.2 | Экран прогноза: горизонт N дней; ветер, волны при наличии API, температура, осадки, давление | UI |
| 4.3 | Модуль приливов: координаты станции NOAA / WorldTides / региональный провайдер | Экран или секция tides |
| 4.4 | Растровый или векторный оверлей погодных полей на карте с прозрачностью и шагом времени | Слой карты |
| 4.5 | Фоновые уведомления при порогах ветра/волны (учесть лимиты iOS/Android) | Опционально |

---

## Детализация

### 4.1 Offline-first

- При отсутствии сети показывать последний успешный прогноз с меткой времени загрузки.
- Не блокировать карту при ошибке API.

### 4.4 Оверлей

- MVP: упрощённая сетка стрелок ветра или цветового поля; производительность на средних устройствах.

---

## Критерии приёмки

- После успешной загрузки данные доступны офлайн до истечения TTL.
- Приливы показывают хотя бы высоты/времена для одной выбранной точки MVP.
- Оверлей не делает карту непригодной для навигации при отключении слоя пользователем за один жест.

## Риски

| Риск | Митигация |
|------|-----------|

| Push в фоне | Чёткая настройка пользователя; fallback только in-app алерты |
