# Фаза 3 — AIS и NMEA

**Цель:** приём данных AIS через NMEA 0183 (TCP/UDP/Bluetooth/serial — по платформе), отображение целей на карте, расчёт CPA/TCPA и предупреждения о сближении.

**Зависимости:** [Фаза 1](phase-01-navigation-core.md); слои карты из [Фазы 2](phase-02-bathymetry-layers.md) желательны для контекста.

**Ссылки:** **[`../Подробный план реализации.md`](../Подробный%20план%20реализации.md)** (§7 NMEA, модуль M7, CPA 100% тесты).

---

## Связь с финальным планом

- **TDD:** **100%** покрытие веток для парсера NMEA/AIS, контрольных сумм и **CPA/TCPA** (фикстуры `test/fixtures/nmea/*.log`).
- **Аудит:** `nmea_connect`, `nmea_disconnect`, изменение порогов CPA алерта; severity `warning` при потере потока.
- **Логирование:** не логировать сырые строки NMEA в production на уровне debug без флага разработчика.

## Детализация

### 3.1 Транспорт

- Переподключение при обрыве Wi‑Fi к шлюзу AIS.
- Буферизация неполных строк NMEA.

### 3.4 CPA/TCPA

- Частота пересчёта согласована с частотой обновления своей позиции и целей.
- Юнит-тесты на синтетических траекториях.

---

## Критерии приёмки

- Запись реального или тестового NMEA-файла воспроизводится с ожидаемыми позициями целей.
- При большом числе целей (>100) карта остаётся отзывчивой (упрощение отрисовки/clustering при необходимости).
- CPA/TCPA совпадают с эталонным калькулятором на тест-кейсах.
